# 隐藏的软件宝石

这一周，我想写写那些隐藏的软件宝石。我的第一个例子是 NFS 服务器。当你在互联网上搜索 NFS server 和 Linux 时，几乎不可避免地会读到集成在 Linux 内核中的 kernel-NFS server。但其实还有第二种实现，只是它受到的关注少得多：NFS-Ganesha 用户态 NFS 服务器。

我的另一个例子当然也来自存储领域：iSCSI target 实现。过去我们有过不少这样的实现，不过其中大多数如今只剩下历史意义。在这里，搜索结果的第一项几乎也总会是内核的 LIO target。但在我看来，真正默默发光的英雄是 SCST target。

我曾经以很痛苦的方式了解到，当你尝试在 VMware hypervisor 作为 iSCSI initiator 的场景中使用 LIO target 时，会暴露出多少问题。我见过它因为内核锁死、iSCSI 会话中止以及更多我为了心理健康已经选择性压抑掉的问题而失败。那真是一团糟。在一次会议上，我遇到了当前的维护者，他非常坦率地说，他们正在尽最大努力让项目继续存活，同时尽可能少地触碰代码，因为修改某一处可能会无意中在其他地方引发问题。这种做法在初始开发者已经离开的项目中很典型。

而那颗宝石，也就是 SCST target，在同样的工作负载下展现出了自己的光芒。这也是我们在 LINSTOR gateway 中默认使用它的原因。

LINSTOR gateway 也提供 NFS export。到目前为止，它们一直基于主流的 kernel-NFS server。我们正在切换到 NFS-Ganesha，以便让单个 LINSTOR gateway 安装能够提供多个高可用 NFS export。
