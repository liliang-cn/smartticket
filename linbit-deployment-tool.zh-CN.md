# 一切始于一个梦想和愿景

这一切始于一个梦想和愿景。上个月，我谈到 AI 如何帮助我把一个梦想变成现实：用一种更简单的方式部署 LINBIT 软件。经过大量测试和反复斟酌后，我很高兴地宣布我们的 LINBIT 部署工具正式推出。对那些还记得项目最初工作代号的内部同事来说，它也叫做“Nightfury”。虽然这个部署工具的愿景来自我，但幕后真正完成工作的，是 LINBIT 团队，尤其是 Matt 和 Ryan，他们开发了该工具调用的 Ansible playbook，用来部署各种不同场景。除了这期通讯之外，我也会在下一次 LINBIT Community Meeting 的 YouTube Live 上介绍这款软件。

这个部署工具的目标，是让你只需要定义自己想要什么，而不必知道具体该如何一步步实现。它大幅降低了入门门槛，也减少了开始测试和部署我们软件所需的时间。现在，用户只需要对目标基础设施有基本了解，就能轻松、快速地部署高可用 SDS 集群。你再也不需要花费数小时学习和阅读文档，虽然我们的文档确实是业内最好的之一，而且闲暇时读一读也很不错。截至今天，这个部署工具支持部署基础 LINSTOR 集群，可以是不带任何服务的集群，也可以部署 NFS、NVMEoF 和 iSCSI 等独立服务，或者按你的选择组合多种服务。

目前，这个部署工具支持以下目标操作系统：Debian 13+、Ubuntu 24.04+、RHEL 9 和 10（以及其衍生版本）。部署工具本身可以运行在 Apple Silicon、Linux 或 Windows 上。虽然你可以从 Windows 系统运行该工具，但 Windows 目前还不支持作为目标系统。不过有人悄悄告诉我，这件事迟早会发生。如果大家对此有兴趣，请告诉我们，因为优先级是可以调整的。

发布前最耗时间的部分，就是测试。要覆盖每一种受支持的操作系统、每一种部署组合，并通过五种不同的安装后故障转移场景进行验证，需要时间，也需要仔细思考。在内部，我们有人使用 SSH 跳板机、本地虚拟机、真实硬件、云服务提供商以及其他环境进行测试，而每一种环境都有自己的注意事项。以我手头的硬件来说，完整跑完一遍测试矩阵通常需要大约 12 个小时。

不过我认为测试中最有意思的部分，是安装后的测试。也就是那个问题：“它在真实世界里扛得住吗？”为了回答这个问题，我收集了工程师和开发人员认为所有集群都应该经历的各种测试场景。测试如下：

1. 计划驱逐：在当前活跃的 Primary 上运行 `drbd-reactorctl evict --force`。这个测试用于确保资源能够平滑迁移，并且某个对等节点能在驱逐延迟时间内成功接管 VIP。
2. 进程终止：在 Primary 上运行 `kill -9 drbd-reactor` 和 `systemctl stop drbd-services@<res>.target`。这个测试用于确认 DRBD 能正确降级，某个 Secondary 节点会提升为 Primary，并且原节点在重启后能干净地重新加入集群。
3. 强制重启：先执行 `poweroff -f`，然后刻意等待 60 秒，以模拟真实硬件的 POST 过程。这里的时间控制非常关键：虚拟机通常重启得比 DRBD 检测到对等节点失效还要快，如果没有这段延迟，原节点可能会在故障转移触发之前重新夺回 Primary，这会掩盖真实世界中的 bug。
4. 网络分区：在每条对等链路上使用 `iptables DROP`。这个测试用于验证基于 quorum 的 fencing 能正确隔离被分区的节点，并且存活的多数节点能够选举出新的 Primary。随后通过简单执行 `iptables -F` 来测试恢复流程。
5. 组合/级联故障：测试控制器和一个 NFS 网关 Primary 同时故障的场景。随后测试会终止第二个 NFS Primary，以刻意破坏 quorum 并迫使 VIP 下线。当节点恢复后，集群无需任何人工干预就能自动恢复 VIP，则测试通过。

在上述每一种场景中，测试都会验证 VIP 是否真的能从新节点响应。仅仅看到 systemd unit 显示绿色，并不足以判定测试通过。

以上就是我在本职工作之外，最近一直在做的一小部分事情。我很希望你能试用这款软件，并提供反馈。作为初始版本，我相信一定还有某些地方被我遗漏了（当然我希望没有！）。你可以在这里下载部署工具。不过需要说明的是：你需要一个 LINBIT 评估账号才能开始使用。这个账号是免费的，只是让 LINBIT 团队可以与你保持联系，以便获取反馈或提供支持。

另外，我也应该提一下，LINBIT Ansible collection 正是这个部署工具背后的魔法所在。你并不需要了解它也能使用这个工具，毕竟这正是部署工具存在的意义。但如果你感兴趣，LINBIT Ansible collection 是一套非常强大的工具，可用于部署和管理 LINBIT 软件。
