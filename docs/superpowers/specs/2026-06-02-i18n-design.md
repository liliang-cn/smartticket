# i18n / l10n 设计方案 — SmartTicket Web 前端

日期: 2026-06-02
范围: `web/` React 前端
状态: 已批准设计,待实现计划

## 目标

为 SmartTicket 控制台前端加入多语言与本地化支持,覆盖 7 种语言:

| 代码 | 语言 |
|------|------|
| `en` | English(兜底语言) |
| `zh` | 中文 |
| `es` | Español |
| `de` | Deutsch |
| `ja` | 日本語 |
| `ko` | 한국어 |
| `it` | Italiano |

## 关键决策(已确认)

1. **纯前端方案** — 后端零改动。后端时间已是 Unix 时间戳、枚举已是稳定码值(`open` / `high` / `within`),不产出需要翻译的展示文本。后端通知/时间线事件/API 错误的英文散文本期不处理(可后续单独立项)。
2. **技术选型** — `react-i18next` + `i18next` + `i18next-browser-languagedetector`。
3. **默认语言** — 浏览器语言检测,命中 7 种之一则用之,否则回退 `en`。用户手动切换后写入 localStorage 键 `st.lang`(沿用现有 `st.theme` 习惯)。
4. **切换器位置** — 顶栏(`app-shell.tsx`,紧邻 `ThemeToggle`)、设置页(`settings.tsx`)、登录页(`login.tsx`)三处。
5. **时区** — 自动跟随浏览器本地时区,不引入时区库。基于原生 `Intl`,12/24 小时制与日期顺序随 locale 自动变化。
6. **交付节奏** — 分阶段:第一阶段搭基础设施 + 核心高频页;其余页面逐个抽取翻译。
7. **翻译来源** — 6 种非英语文案由实现方按 `en` 资源直接生成,专业术语(SLA、工单状态)采用行业惯用译法。

## 架构

### 初始化层 `src/lib/i18n.ts`

- 新建模块,配置 i18next 实例:`fallbackLng: "en"`、`supportedLngs` 为上述 7 种、`detection` 顺序为 `localStorage`(键 `st.lang`)→ `navigator`。
- 在 `main.tsx` **最外层**引入(早于 `ThemeProvider` / `AuthProvider` 等,因为 Provider 内部也需要翻译)。
- 切换语言时:调用 `i18next.changeLanguage(lng)`、写入 `st.lang`、更新 `document.documentElement.lang`。
- 资源以静态 import 方式按 namespace 装载(体量可控,暂不做按需懒加载;若后续打包过大再切 lazy)。

### 翻译资源 `src/locales/`

```
src/locales/
  en/
    common.json        # 导航、按钮、表单通用词、枚举码值映射(enums.*)
    auth.json          # 登录页
    tickets.json
    knowledge.json
    customers.json
    users.json
    products.json
    services.json
    subscriptions.json
    sla.json
    llm.json
    data.json          # 导入导出 / data jobs
    settings.json
    notifications.json
  zh/ es/ de/ ja/ ko/ it/   # 与 en 同结构
```

- namespace 切分对齐现有 `src/features/*` 划分。
- `common.json` 收纳共享文案与 **枚举码值→标签** 的映射(`enums.ticket_status.open`、`enums.priority.high` 等)。
- 提供 `useEnumLabel(group, code)` 辅助函数,组件内统一把后端码值翻成当前语言标签。

### 日期 / 时间本地化 `src/lib/utils.ts`

收敛分散的时间格式化逻辑为统一 helper,全部读取 i18next 当前 locale 喂给原生 `Intl`:

- `formatDateTime(ts)` → `Intl.DateTimeFormat(locale, { dateStyle: "medium", timeStyle: "short" })`
- `formatDate(ts)` → `Intl.DateTimeFormat(locale, { dateStyle: "medium" })`
- `relativeTime(ts)` → 改写现有实现,locale 跟随当前语言(替换写死的 `"en"` 与英文 `"just now"`),相对时间单位词改用 `Intl.RelativeTimeFormat(locale)` + i18n 文案。
- helper 接受 Unix 秒(`number`)与 ISO 字符串两种输入(兼容现有调用点)。
- 收编现有各自为政的格式化点:`src/components/notification-bell.tsx`、`src/pages/subscriptions-list.tsx`。
- 时区由 `Intl` 默认取浏览器本地时区,自动正确,无需配置。

### 语言切换器 `LanguageToggle`

- 新增组件,复刻 `app-shell.tsx` 中 `ThemeToggle` 的 `DropdownMenu` 模式:下拉列出 7 种语言(各自原生名称),当前项高亮。
- 放置点:
  - 顶栏 `app-shell.tsx`,紧邻 `<ThemeToggle />`。
  - 设置页 `settings.tsx`,作为一个 section。
  - 登录页 `login.tsx`,登录前也可切换。
- 选择即时生效并持久化。

## 数据流

```
浏览器/ localStorage(st.lang)
        │  detection
        ▼
   i18next 实例 ──► useTranslation(ns) ──► t("key") ──► 组件渲染
        │                                   ▲
        │ changeLanguage(lng)               │
        ▼                                   │
  LanguageToggle ──► 写 st.lang / <html lang>
        │
        └──► 当前 locale ──► Intl 日期 helper ──► 本地化时间/日期/相对时间
```

## 落地范围与方式

- 组件内将硬编码英文文案抽取到对应 namespace 资源文件,改用 `const { t } = useTranslation(ns)` → `t("key")`。
- 先把 `en` 资源做全(直接从现有英文文案抽取,保证零回归),再据此生成其余 6 语。
- **分阶段交付**:
  - **阶段一(基础设施 + 核心页)**:`i18n.ts` 初始化、`LanguageToggle`(三处)、日期 helper 收敛、`common` namespace、`auth`/导航/`dashboard`/`settings` 抽取与 7 语翻译。框架端到端跑通、可切换、可验证。
  - **阶段二及以后**:按 feature 逐页抽取 + 翻译(tickets → knowledge → customers → users → products/services → subscriptions → sla → llm → data → notifications)。

## 错误处理 / 边界

- 缺失 key:i18next 默认回退 `en`;开发模式开启 `saveMissing` 警告以便发现漏抽。
- 不支持的浏览器语言:回退 `en`。
- `st.lang` 存了非法值:`supportedLngs` 校验后回退 `en`。
- 枚举码值映射缺失:`useEnumLabel` 兜底返回原始码值,避免渲染空白。

## 测试

- **i18n 配置单测**:浏览器语言检测命中/未命中(回退 `en`);`changeLanguage` 持久化到 `st.lang`;`<html lang>` 同步更新。
- **日期 helper 单测**:同一时间戳在 `en` / `de` / `ja` 下输出格式不同且符合各 locale 习惯(12/24 小时、日期顺序)。
- **资源完整性脚本**:校验 6 种语言每个 namespace 的 key 集合与 `en` 完全一致,防漏翻 / key 不对齐;接入 lint/CI。

## 不在本期范围(YAGNI)

- 后端通知 / 时间线事件 / API 错误文本的多语言(后续单独立项,需改后端为「码值 + 参数」结构)。
- 历史 DB 记录(已写入的英文通知/事件)的回溯翻译。
- 手动时区选择器(自动跟随浏览器已覆盖 99% 场景)。
- 翻译资源的运行时懒加载 / 按需分包(待打包体量过大再做)。
- RTL 语言(本期 7 语均为 LTR)。
